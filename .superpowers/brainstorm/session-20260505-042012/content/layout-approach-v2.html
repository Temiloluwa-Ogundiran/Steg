Approach B: Vertical Node Dial

Modes selected via vertical step indicator on the left. Main area is wider. Terminal sidebar remains.

Pros

  • More distinctive, follows template's "DNA" section closely
  • Vertical hierarchy feels like a pipeline

Cons

  • Takes more horizontal space
  • Less immediately scannable than a grid

Approach C: Dashboard Grid (All Visible)

All three mode forms visible simultaneously. Active mode highlighted, others dimmed. Terminal sidebar remains.

Pros

  • No mode switching needed
  • Full density

Cons

  • Cluttered — three file inputs visible at once
  • Confusing which submit button to press

My recommendation: Approach A — it's the cleanest mapping of the template to our 3-mode workflow. The segmented grid is instantly understandable, and the terminal sidebar gives us that developer-aesthetic result panel you wanted.
